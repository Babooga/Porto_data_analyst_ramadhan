# 🏥 Patient Waitlist Breakdown Dashboard

## Overview
I came across a fascinating YouTube tutorial on healthcare data visualization, 
which inspired me to build this dashboard as a hands-on exploration of 
real-world analytics in the medical sector.

This **Patient Waitlist Breakdown Dashboard** was developed using Power BI to deliver 
a real-time and historical view of patient waitlist dynamics. Designed for healthcare 
administrators and analysts, it provides a structured way to monitor patient flow, 
detect bottlenecks, and support smarter resource allocation — transforming raw 
medical records into clear, actionable insights.

## Objective
The goal of this project is to analyze and monitor hospital patient waitlist performance 
across multiple dimensions — including case types, age profiles, and admission categories. 
This dashboard enables healthcare stakeholders to track waitlist trends over time, compare 
current performance against previous periods, and make informed decisions to improve 
patient care efficiency.

---

## Dashboard Features

### 📌 Key Metrics (KPI Cards)
* **Latest Month Waitlist** — Total number of patients currently on the waitlist 
as of the most recent month
* **Key Indicator — Average Patient Waitlist** — A core performance indicator 
reflecting the average number of patients on the waitlist across the observed period
* **PY Latest Month Waitlist** — Previous year's waitlist figure for the latest 
month, enabling year-over-year performance comparison

### 📊 Analytical Visualizations
* **Average Waitlist by Case Type** — Breakdown of average waiting patients 
segmented by case type (Day Case, Inpatient, Outpatient)
* **Age Profile Analysis** — Distribution of waitlist patients across different 
age groups, providing demographic insight into patient demand
* **Monthly Trend Analysis** — Time-series visualization comparing Day Case / 
Inpatient trends versus Outpatient trends month over month

### 🔍 Interactive Features
* **Filter Criteria** — Dynamic filtering options allowing users to slice data 
by date range, case type, specialty, and other dimensions for focused analysis
* **Detail View** — A granular data table providing an in-depth breakdown of 
individual patient waitlist records for deeper investigation
